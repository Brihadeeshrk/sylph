# Headings

To create a heading, add one to two <kbd>#</kbd> symbols before your heading text. The number of <kbd>#</kbd> you use will determine the hierarchy level and typeface size of the heading.

<Preview codeblock>
  <div>
    <h1>Heading 1</h1>
    <div/>
    <h2>Heading 2</h2>
  </div>
</Preview>

```markdown
# Heading 1
# Heading 2
```

# Styling Text

You can indicate emphasis with bold, italic, strikethrough, subscript, or superscript text in comment fields and .md files.

<Preview codeblock>
  <div>**Bold**</div>
  <div>*Italic*</div>
  <div>~~Strikethrough~~</div>
  <div><sub>Subscript</sub></div>
  <div><sup>Superscript</sup></div>
  <div><kbd>⌘</kbd><kbd>K</kbd></div>
</Preview>

```markdown
**Bold**
*Italic*
~~Strikethrough~~
<sub>Subscript</sub>
<sup>Superscript</sup>
<kbd>⌘</kbd><kbd>K</kbd>
```

# Quoting Text

You can quote text with a <kbd>></kbd>.

<Preview codeblock>
  Text that is not a quote
  > Text that is a quote
</Preview>
```markdown
Text that is not a quote
> Text that is a quote
```

# Links

You can create an inline link by wrapping link text in brackets <kbd>[ ]</kbd>, and then wrapping the URL in parentheses <kbd>( )</kbd>.

<Preview codeblock>
  [Link](https://www.youtube.com/watch?v=dQw4w9WgXcQ)
</Preview>
```markdown
[Link](https://www.youtube.com/watch?v=dQw4w9WgXcQ)
```

# Lists

You can make an unordered list by preceding one or more lines of text with <kbd>-</kbd>.

<Preview codeblock>
  - George Washington
  - John Adams
  - Thomas Jefferson
</Preview>
```markdown
- George Washington
- John Adams
- Thomas Jefferson
```
To order your list, precede each line with a number.

<Preview codeblock>
  1. James Madison
  2. James Monroe
  3. John Quincy Adams
</Preview>
```markdown
1. James Madison
2. James Monroe
3. John Quincy Adams
```

# Footnotes

You can add footnotes to your content by using this bracket syntax <kbd>`[^1]`</kbd> and then defining the footnote at the bottom of your content.


<Preview codeblock>
  <div>Here is a simple footnote[^1].</div>
  <div>A footnote can also have multiple lines[^2].</div>
  [^1]: My reference.
  [^2]: To add line breaks within a footnote, prefix new lines with 2 spaces.
    This is a second line.
</Preview>
```markdown
Here is a simple footnote[^1].
A footnote can also have multiple lines[^2].

[^1]: My reference.
[^2]: To add line breaks within a footnote, prefix new lines with 2 spaces. This is a second line.
```

# Preview 

You can preview your markdown content in real-time by using the preview feature in the editor. We have been using it throughout this guide to show you how your markdown content will look when rendered. It comes with a single prop, <kbd>codeblock</kbd>, that you can use to display the code block right below the preview.

# Inline Code

Append <kbd>`{:lang}{:js}`</kbd> (e.g. <kbd>`{:js}{:js}`</kbd>) to the end of inline code to highlight it like a regular code block.

<Preview codeblock>
  To useEffect or not to `useEffect(){:js}`, that is the question.
</Preview>
```markdown
This is an array `[1, 2, 3]{:js}` of numbers 1 through 3.
```


# Code Blocks

To format code or text into its own distinct block, use triple backticks.
 
<Preview codeblock>

  return "It works in my environment.";
}
```
</Preview>
```markdown
```tsx
